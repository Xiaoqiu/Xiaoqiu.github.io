Sed - Overview

  • The acronym SED stands for Stream EDitor.
  • Typical Uses of SED

  • SED can be used in many different ways, such as:

  • Text substitution,

  • Selective printing of text files,
  • In-a-place editing of text files,
  • Non-interactive editing of text files, and many more.

Sed - Environment

Sed - Workflow

SED follows a simple workflow: Read, Execute, and Display.

  • Read: SED reads a line from the input stream (file, pipe, or stdin) and stores it in its internal buffer called pattern buffer.

  • Execute: All SED commands are applied sequentially on the pattern buffer. By default, SED commands are applied on all lines (globally) unless line addressing is specified.

  • Display: Send the (modified) contents to the output stream. After sending the data, the pattern buffer will be empty.

The above process repeats until the file is exhausted.

  • sed命令作用于缓存里面的文本，然后输出并删除缓存。

  • [address1[,address2]]y/list-1/list-2/
  • 转换是根据字符位置的。list1的字符转换为list2对应位置的字符。
  • 正则，字符类别是不支持的
  • list1的大小必须和list2一样

l command

  • l 命令显示隐藏的字符，例如tad, spaces
  • tad字符是\t， 结束字符是$
  • [address1[,address2]]l
  • [address1[,address2]]l [len]
  • it also accepts line numbers and patterns as an address.

Quit Command

  • 中断执行流
  • 一般sed跟read, execute, repeat 工作流，当遇到quit命令，停止当前执行。
  • 不跟地址范围，只支持一个地址。

Read Command

  • [address]r file
  • r 后面必须有一个空格
  • 可以接受地址范围

Execute Command

  • 可以用来执行外部命令
  • [address1[,address2]]e [command]
  • 接受模式匹配作为地址
  • command参数是可选的。如果e后面不带command的，就会把缓存内容作为外部命令
  • 缓存内容就是文件内容

  • [/pattern/]=
  • [address1[,address2]]=

& Command

  • 匹配字符成功后，&保存匹配的内容。&作为一个占位符后面使用。
