部署

https://downloads.dcos.io/dcos/stable/dcos_generate_config.sh
1 创建安装文件
bash dcos_generate_config.sh –genconf
2 安装dcos需要的依赖环境
bash dcos_generate_config.sh –install-prereqs
3 检测各节点是否已达到了安装部署dcos的全部要求
bash dcos_generate_config.sh –preflight

4 正式安装,执行deploy，其实就是从boot节点把安装镜像下发到各个节点而已，上一步没问题这一步一般也不会出错。运行成功后证明各机器已经成功安装dcos
bash dcos_generate_config.sh –deploy

5 确保服务启动并且正常运行，这步其实就是boot去各个节点把docker镜像拉起来，DCOS的那些组件像mesos master 、slave 、marathon、mesos-dns等都是运行在docker里，另外具体配置信息也是boot节点通过环境变量这种方式帮你注入进去，这一步一般也不会出现问题。运行成功后dcos服务已经启动
bash dcos_generate_config.sh –postflight

6 dcos服务监控 监控dcos服务的网址为http://:8181/exhibitor/v1/ui/index.html，

7 登陆DC/OS 登陆DC/OS系统的接口为http:///

添加agent节点

1 安装依赖
因为dcos节点在部署之前需要安装一些以来文件，这里的做法是将需要动态添加到集群中的节点的ip地址写入
genconf/config.yaml文件的响应位置，并
运行bash dcos_generate_config.sh –install-prereqs命令安装依赖。
这里因为集群中原有集群中已经完整安装过一次依赖，所以即使它们的ip地址也在config.yaml文件中，但该步也只会向新加入的节点中安装
1.1 # 检测各节点是否已达到了安装部署dcos的全部要求
bash dcos_generate_config.sh –preflight

2 在boot节点生成dcos-install.tar
cd genconf/serve
tar cf dcos-install.tar *

3 复制安装文件
scp dcos-install.tar root@192.168.33.14:/home/dcos/

4 进行安装
4.1 到docs8-130机器上将dcos-install.tar文件解压
mkdir -p /opt/dcos_install_tmp & tar xf /home/dcos/dcos-install.tar -C /opt/dcos_install_tmp
4.2 执行安装
bash /opt/dcos_install_tmp/dcos_install.sh slave
bash /opt/dcos_install_tmp/dcos_install.sh master

4、安装DC/OS在新增节点
增加private节点
$ sudo bash /opt/dcos_install_tmp/dcos_install.sh slave

增加public节点
$ sudo bash /opt/dcos_install_tmp/dcos_install.sh slave_public

5、验证
查询private节点个数
dcos node –json | jq –raw-output ‘.[] | select(.reserved_resources.slave_public == null) | .id’ | wc -l
查询public节点个数
dcos node –json | jq –raw-output ‘.[] | select(.reserved_resources.slave_public != null) | .id’ | wc -l
